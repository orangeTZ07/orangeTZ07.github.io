今天我们来迈进rust。本文档中并不会去介绍rust的语法之类的基础知识，而是会记录自己对rust的设计理念的理解，以及在使用rust过程中遇到的一些问题和解决方案。
首先，rust的设计理念是安全、并发和性能。rust通过所有权系统来保证内存安全，避免了传统的内存管理问题，如空指针、悬空指针和数据竞争。同时，rust还提供了强大的并发支持，使得开发者能够轻松地编写高性能的并发程序。
这一点是大家共识的。那我们就从语法和设计讲起，看看rust究竟安全在哪。

1.rust变量
```
let a = 1;
a = 2; //error
------
let a = 1;
let a = 2;
let mut a = a;
a = a+1;
```
这种不可变变量和可变变量区分开的好处在哪呢？
```
//进入可能产生危险的代码块，威胁在于黑客可能修改a的值
let a = 1;
//要输入修改a的值，同时让后续代码能够更方便沿用a
let mut a = a;
a = input();
//进入可能产生危险的代码块，威胁在于黑客可能修改a的值
let a = a;//让a重新变得不可修改

```
